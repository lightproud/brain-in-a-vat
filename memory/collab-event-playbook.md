## 响应分三阶段

### 阶段 1：联动公布时（T+0 ~ T+24h）

**目标**：捕获社区第一反应

**自动化任务（日报系统应做到的）**：
- Steam 评论中出现联动关键词时自动标记
- Discord 消息量峰值检测（如果当日消息 > 均值 2 倍，日报标注"异常活跃"）
- Bilibili 搜索命中联动关键词的视频/动态

**人工任务（制作人/运营）**：
- 在 Discord 社区观察第一反应情绪
- Reddit 相关帖子截图/链接留存

**Wiki 任务（Code-wiki）**：
- 准备联动角色页面模板（基于现有角色页结构）
- 联动作品背景介绍页（一页，中英日三语）

### 阶段 2：联动上线时（版本更新日）

**目标**：快速产出内容 + 持续监测社区反馈

**Wiki 任务**：
- 联动角色数据录入（技能、数值、立绘）
- 联动关卡/活动数据录入
- 联动获取方式说明页

**日报任务**：
- 连续 7 天密切监测各平台反馈
- 特别关注：Steam 好评率变化、联动角色强度评价、"抽不到"相关负面情绪

### 阶段 3：联动结束后（T+30天）

**目标**：沉淀数据 + 复盘

**银芯任务**：
- 联动期间社区数据汇总（消息量趋势、情绪变化、热门话题）
- 事实圣经更新：联动角色信息、联动剧情摘要

**黑池任务**（如已启动）：
- 联动付费数据分析
- 联动角色使用率/胜率
- 复盘文档

## 聚合器关键词预配置

联动作品公布后，需在 aggregator.py 的搜索关键词中追加：
- 联动作品名（中英日）
- 联动角色名（中英日）
- "联动"/"collab"/"collaboration"/"コラボ"

当前 aggregator.py 的搜索关键词：
- 中文：忘却前夜、忘卻前夜
- 英文：Morimens、morimens

**建议**：在 aggregator.py 中增加一个 `EXTRA_KEYWORDS` 配置项（或环境变量），联动时可通过 GitHub Secrets 快速追加关键词，不需要改代码。
