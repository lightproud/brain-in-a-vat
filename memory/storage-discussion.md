# 大文件存储方案讨论

> 最后更新：2026-03-29 by Code-news

## 背景

项目中有两类大文件存储需求：

### 1. Discord 全量归档（News 子项目）
- 服务器所有频道消息的附件（图片、视频、文件）
- Discord CDN URL 会过期，需要下载保存
- 预估：每天 50-200 张图 + 少量视频 = **100-500MB/天**

### 2. Wiki 游戏资产（Wiki 子项目）
- 角色立绘、技能图标、装备图片、UI 素材
- 相对固定，但总量可能较大
- 预估：初始 **500MB-2GB**，后续随版本更新增长

## 方案对比

| 方案 | 费用 | 容量 | 优点 | 缺点 |
|------|------|------|------|------|
| **Git LFS** | 免费 1GB，超出 $5/50GB/月 | 无上限（付费） | GitHub 原生集成，`git push` 直接用 | 费用随量增长，clone 仍然慢 |
| **GitHub Releases** | 免费 | 单文件 2GB，总量无明确限制 | 免费，CDN 分发快 | 需要打包上传，不方便检索单文件 |
| **Cloudflare R2** | 免费 10GB，超出 $0.015/GB/月 | 无上限 | 免费额度大，无出站流量费，S3 兼容 | 需要 Cloudflare 账号，额外配置 |
| **阿里云 OSS** | ~¥0.12/GB/月 | 无上限 | 国内访问快，按量付费 | 需要实名认证 |

## 需要讨论的问题

1. **存储方案选择**：统一用一种，还是 Discord 归档和 Wiki 资产分开处理？
2. **Discord 附件**：是否需要全量下载保存？还是只保存文本数据，附件按需获取？
3. **Wiki 资产**：图片资源是否需要版本管理？还是只保留最新版本？
4. **预算**：是否接受付费方案？如果是，月预算大概多少？
5. **访问方式**：资产是否需要公开 CDN 访问（比如 wiki 站点直接引用）？

## 建议

- **Wiki 资产**：量相对固定且需要公开访问 → **Cloudflare R2** 或 **Git LFS** 都适合
- **Discord 归档**：量大且持续增长 → **Cloudflare R2** 性价比最高，或者 **GitHub Releases** 按周打包
- 如果想统一方案，**Cloudflare R2** 最灵活，两边都能用
