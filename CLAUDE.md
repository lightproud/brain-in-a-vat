# 忘却前夜 Morimens — 项目总控制

## 项目愿景

收集忘却前夜（Morimens）的各方面情报、数据和资产，最终制作一款衍生同人游戏。

本项目同时承载两层目标：
- **方法论层**：验证并沉淀"AI 多会话协作开发模式"，形成可复制的方法论，面向所有 AI 开发者开源
- **内容层**：构建忘却前夜的开放数据平台，鼓励游戏社区共同贡献和使用数据

## 授权与数据原则

- **项目所有者为忘却前夜官方授权制作人**，拥有游戏资产的完整使用权限
- 游戏资产（立绘、模型、音频等）和结构化数据均可自由使用，无版权限制
- 游戏设计内容、系统内容、资产内容归属**脑缸组及其合作伙伴**所有，本项目引用公开可查阅信息
- 仓库公开，欢迎社区参与共建
- 方法论文档（`docs/`）面向所有 AI 开发者，与游戏内容无关也可复用

## 架构说明

本项目采用 **Claude Code 多会话协作模式**：
- 每个 Claude Code 对话 = 一个独立子项目
- **主控制台**（本文件所在会话）负责统一协调
- 各子项目通过本文件 (`CLAUDE.md`) 同步状态和共享决策
- **新会话启动时必须先读取本文件，了解项目全局状态**

## 协作规范

### 分支管理
- 分支命名：`claude/<功能描述>-<ID>`
- 各子项目在自己的分支上开发
- 由主控制台决定何时合并

### 代码风格
- 前端：纯 HTML/CSS/JS，无框架
- 后端：Python 3.11+
- 部署：GitHub Pages + GitHub Actions

### 给新会话的指引
如果你是一个新启动的 Claude Code 会话，请：
1. 先通读本文件，了解项目全局
2. 确认你负责的子项目和当前待办
3. 在你的子项目目录下工作
4. 完成重要决策后，更新本文件中对应的子项目状态
5. 不要修改其他子项目的代码
