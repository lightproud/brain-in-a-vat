02
Token 体系说明
所有 Token 以 --m- 前缀命名，避免与 VitePress --vp- 和其他框架变量冲突。

命名规则

前缀含义示例
--m-bg-*背景层级--m-bg-abyss, --m-bg-card
--m-gold-*品牌金色--m-gold-primary, --m-gold-glow
--m-text-*文本色--m-text-primary, --m-text-dim
--m-realm-*界域辅助色--m-realm-caro
--m-border-*边框--m-border-hover
--m-font-*字体族--m-font-title
--m-fs-*字号--m-fs-hero
--m-sp-*间距--m-sp-8 = 32px
--m-dur-*动画时长--m-dur-drift = 12s

色彩比例铁律

85 / 10 / 5 — 深色底 85% · 暖灰文本 10% · 金色点缀 5%。任何新组件在动手前先过这个比例检查。

氛围层三件套

Token 文件提供了三个工具类：.m-fog（雾气）、.m-grain（噪点）、.m-vignette（暗角）。使用时在容器上添加 position: relative; overflow: hidden; 即可叠加。三层可以组合使用，也可以单独使用。

性能红线：雾气动画只用 transform + opacity，不触发 layout。噪点是 SVG repeat，不用 canvas。三层都用 pointer-events: none 避免阻断交互。

  extends: DefaultTheme,
  Layout() {

      'doc-after': () => h(GiscusComments),
    })
  },
  enhanceApp({ app }) {
    // ... 组件注册保持不变 ...
  },
}

游戏组件样式迁移

原 custom.css 中的游戏组件样式（.rarity-*、.realm-badge、.card-grid 等）已全部迁移到新的主题 CSS 中，并改用 Token 变量。所有 Vue 组件无需修改模板，只要 class name 没变就自动生效。

不兼容注意：新 CSS 中 .card 类增加了 background: var(--m-bg-surface)，如果现有 Vue 组件中有内联 background 样式，需要删除内联让 CSS 接管。

载体 C · 社区情报日报
日报已基本对齐深渊+金色体系。需要做的是变量命名对齐和微调。

当前状态 vs 目标

项目当前值对齐目标差异

Gold#c5a356--m-gold-primary✓ 一致
Text#d4c4a0#f0ebe0▲ 需调亮
Title fontNoto Serif SC--m-font-title✓ 一致
Body fontNoto Sans SC--m-font-body✓ 一致
Divider#2a2520rgba(gold, 0.10)▲ 需换用金色透明度

适配方式

日报模板是内联 CSS（因为要生成 PDF），不适合直接引用外部 Token 文件。做法是：

/* 在日报模板 <style> 块开头，手动同步核心 Token */
:root {
  --bg: #0a0b10;  /* = --m-bg-abyss */
  --gold: #c5a356;  /* = --m-gold-primary */
  --txt: #f0ebe0;  /* = --m-text-primary  ← 从 #d4c4a0 调亮 */
  --txt-d: #b5aa98;  /* = --m-text-muted */
  --bdr: rgba(197, 163, 86, 0.10);  /* = --m-border-default */
  /* ... */
}
PDF 渲染限制：weasyprint 不支持 backdrop-filter、CSS 动画、SVG filter 噪点纹理。日报中的氛围层只能用纯 radial-gradient 实现，不叠噪点和动画。这是 PDF 载体的固有约束，不影响视觉统一性。
06
设计护栏
任何人（包括 AI agent）在三个载体上新增视觉元素时，必须遵守的硬性约束。

绝对禁止

禁止项原因
大面积金色铺陈金色是「记忆的残光」，出现得少才有意义。5% 上限
界域色作为纯色填充只能作为雾气层 radial-gradient，≤15% opacity
弹跳/旋转/缩放 >1.02 动画基调是沉重、缓慢、宿命感，不是活泼
白色或亮色背景视觉语言是「从黑暗中浮现」
圆角 >12px锐利感服务于克苏鲁美学
字号 <14px（移动端正文）移动端可读性底线
非体系字体只能用 Noto Serif SC + Noto Sans SC + 等宽码字体

必须遵守

规则说明
骨架对称 · 细节自由布局居中对称，纹理/雾气允许不对称
一屏一主题全幅区块独占至少一个 viewport 高度
标题必须宋体h1–h3 用 --m-font-title，不可用黑体
所有新颜色必须走 Token禁止硬编码色值。新增颜色需先加入 Token 文件
对比度 ≥ 4.5:1WCAG AA 标准。主文本 on 深渊底 = 11.2:1 ✓
prefers-reduced-motion所有动画必须在 reduced-motion 下归零

给 AI Agent 的指令模板
以下文本可直接复制到 Claude Code 或 Chat session 的系统指令中，让 agent 在执行任务时自动遵守设计规范。
## 视觉规范约束（Design System v1.0）

你在为「忘却前夜 Morimens」项目生成前端代码或视觉内容时，
必须遵守以下约束：

### 色彩
- 背景: #0a0b10（深渊）/ #0f1018（表面）/ #161722（抬升）
- 品牌金: #c5a356（主）/ #e2c97e（亮）/ #8a7340（暗）
- 文本: #f0ebe0（主）/ #b5aa98（次）/ #7a7265（弱）
- 金色面积 ≤5%，界域辅助色只用于 radial-gradient 雾气层 ≤15% opacity

### 字体
- 标题: Noto Serif SC SemiBold(600)/Bold(700)
- 正文: Noto Sans SC Medium(500) ≥15px
- 代码: JetBrains Mono
- 移动端最小字号: 14px

### 动效
- 只允许: fade-in, translateY, opacity, transform
- 禁止: bounce, rotate, scale>1.02, spring
- 缓动: cubic-bezier(0.4, 0, 0.2, 1)
- 雾气漂移: 10-15s, ease-in-out, infinite alternate

### 氛围层
- 三层: 雾气(radial-gradient) + 噪点(SVG feTurbulence) + 暗角(radial-gradient)
- 噪点 opacity ≤ 0.05
- 所有层 pointer-events: none

### 布局
- 最大内容宽度: 960px
- 基础间距单位: 8px
- 一屏一主题，区块间距 80-120px
- 骨架对称，细节自由
